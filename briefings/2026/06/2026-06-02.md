# Daily Loop-Model Briefing — 2026-06-02

## Outcome

Five papers were added to the collection. The day is unusually broad: one paper gives theory for looped linear transformers with layer normalization, two papers put explicit transformer-block reuse into language or multimodal generative systems, and two papers apply recursive transformer depth to computer vision systems.

## Added to the collection

### Looped Transformers with Layer Normalization Provably Learn the Power Method

This paper studies principal component prediction as a controlled setting for understanding what a looped transformer learns. The main result is a proof that a looped linear transformer with layer normalization, trained by gradient descent for the prediction task rather than by direct algorithm supervision, converges to a solution that implements the power method.

Why it was added: the paper directly analyzes looped transformers, with self-attention layers reused as power-iteration steps inside the model computation. It belongs in analysis because the main contribution is a theoretical account of training dynamics and layer normalization.

Main reader takeaway: layer normalization is not treated as incidental; the authors prove a gap between looped transformers with and without it for exactly learning the power-method mechanism.

### CART: Context-Anchored Recurrent Transformer

CART reuses one shared transformer core multiple times across depth while anchoring the recurrent core to key-value tensors computed once by a prelude. The paper is valuable partly because its result is negative: under the reported recipe, CART does not beat dense baselines at parameter parity, and several recurrent-core mechanisms appear vestigial in ablations.

Why it was added: even with the negative result, the architecture is explicitly a depth-recurrent transformer that reuses a learned shared core within one model forward process. It belongs in designs because it proposes and evaluates a concrete looped language-model architecture.

Main reader takeaway: weight sharing and recurrent-depth machinery need careful validation; this paper is useful evidence about where a plausible recurrent transformer recipe fails.

### Déjà View: Looping Transformers for Multi-View 3D Reconstruction

DéjàView turns decoder-depth refinement in multi-view 3D reconstruction into an explicit loop. A single looped transformer block is applied recurrently to per-view features for K refinement steps, making the number of loops an inference-time compute knob.

Why it was added: the central mechanism is reuse of one learned transformer block across refinement steps. It belongs in applications because the looped model is tested as a practical architecture for multi-view 3D reconstruction.

Main reader takeaway: the authors report that the looped block matches or outperforms substantially larger feed-forward baselines across five reconstruction benchmarks while using fewer parameters and comparable or lower compute.

### Fixed-Point Masked Generative Modeling

This paper converts part of masked generative model denoisers into a fixed-point module over shared attention layers. The method combines fixed-point denoising with cross-step consistency and reuse of previous solver states, so masked generation can adapt computation depth instead of allocating a fixed denoiser budget everywhere.

Why it was added: masked generation alone would not qualify for this repository, but the central model change is an implicit layer over shared learned attention layers. It belongs in designs because it gives a concrete training-to-inference framework for fixed-point masked generation.

Main reader takeaway: the abstract reports fewer parameters, lower training time and VRAM, and better low-budget generation quality on both OpenWebText and ImageNette.

### Recursive Vision Transformer with Dynamic Depth and Width Adjustment

This paper introduces a recursive ViT for image semantic communication. The model iteratively refines semantic features and uses dynamic depth, dynamic width, and joint width-depth optimization to adapt computation to image content and channel conditions.

Why it was added: the recursive structure is used to reuse learned transformer computation for iterative feature refinement while reducing parameter count. It belongs in applications because the looped transformer mechanism is deployed for resource-efficient image semantic communication.

Main reader takeaway: the reported system reduces parameter count by 48.7 percent and improves reconstruction quality over baselines at comparable compute.

## Checked but not added

- **Fixed Universal Transformers** (`2605.31423`) is a theory paper about fixed transformers simulating other transformers through input embeddings. It does not center on reusing a learned internal layer, block, module, or operator within one forward process.
- **PRISM: Progressive Reasoning through Iterative Slot Memory for Vision** (`2605.30942`) is adjacent because it uses recurrent multi-scale organize-recall-refine processing, but the abstract does not make explicit learned layer or block reuse central enough for the strict scope.
- **Latent Recurrent Transformer** (`2605.26797`) remains not added. Its recurrence carries a high-level source-layer hidden state across token positions while preserving ordinary depth and KV-cache flow, so it is closer to sequence-time recurrent memory than same-forward depth reuse.
