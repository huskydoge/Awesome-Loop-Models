## Added to the collection

### Tying the Loop -- Tied Expert Layers in Mixture-of-Experts Language Models

The paper introduces Expert Tying for MoE language models. Standard MoE LMs spend most of their parameter memory on expert MLPs, and this work ties those expert parameters across consecutive Transformer layers. The tied layers still have independent layer-wise routing and attention, so the design does not collapse the full stack into one repeated block; instead, it selectively reuses the expert submodules that dominate memory.

Why it was added: the repository scope includes weight-tied internal modules reused within one forward process. This paper explicitly shares learned expert parameters across consecutive Transformer layers, and the paper title and code release frame the method as looped MoE.

Main reader takeaway: this is a narrow but useful loop-model design point. It suggests that loop-style parameter reuse can be applied to the expensive MoE expert path without forcing every layer component to be shared. The reported result is roughly a 2x expert-memory reduction with little degradation in perplexity or downstream quality across OLMoE, Qwen3, and DeepSeek-style MoE settings.
