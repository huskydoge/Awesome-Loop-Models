# Daily Loop-Model Briefing — 2026-05-07

## Outcome

Two May 7 arXiv papers were added to the main collection. Both were verified on their exact arXiv abstract pages and both make the shared-operator loop explicit rather than relying on generic “iterative” or “adaptive” wording.

## Added to the collection

### Is One Layer Enough? Understanding Inference Dynamics in Tabular Foundation Models

This ICML 2026 paper studies six tabular foundation models layer by layer and finds substantial depthwise redundancy during inference. The loop-model hook is explicit: the authors use that redundancy to build a proof-of-concept looped single-layer model that keeps comparable performance while using about 20% of the original parameters.

Why it is in scope: the added model reuses a learned layer across depth within one prediction, making it a flat loop over an internal operator. It is categorized as analysis because the main contribution is a mechanistic study of inference dynamics, with the looped model serving as the design probe.

### Transformers Efficiently Perform In-Context Logistic Regression via Normalized Gradient Descent

This theory paper shows that softmax transformers can implement in-context logistic regression, with each layer corresponding to one normalized-gradient-descent step. It then shows that training a single self-attention layer on one-step gradient descent and recurrently applying that layer yields a looped model, with convergence and out-of-distribution generalization guarantees.

Why it is in scope: the model explicitly reuses one trained attention layer repeatedly inside the forward computation. It is categorized as analysis because the main value is the theoretical mechanism result for in-context learning.

## Checked but not added

- **UniPool** shares one global expert pool across transformer layers. This is close to the repo boundary because the experts are shared internal modules, but the abstract frames the contribution as shared MoE capacity and depth-scaling efficiency rather than a looped/refinement block. It stays on the watchlist for now.
- **Dynamic Pondering Sparsity-aware MoE Transformer** adapts inference depth for event-stream tracking, but the abstract does not show a reused loop block.
- **Adaptive Computation Depth via Learned Token Routing** learns token-level residual-update gates over consecutive transformer blocks. This is adaptive routing/skipping, not same-operator recurrence.
- **RD-ViT** is a strong recurrent-depth vision candidate, but it was published on May 5 and surfaced here through exact-phrase/backlog search rather than the May 7 new-paper slice. It should be handled separately from this daily repair PR.
