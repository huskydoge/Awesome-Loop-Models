# Daily Loop-Model Briefing — 2026-05-14

## Outcome

No new paper was added to the main collection. The same-day strict-keyword sweep found one close inverse-problem paper, and the exact-phrase sweep also surfaced several recent cache, latent-state, and forward-refinement papers. Each was checked on its exact arXiv abstract page before being kept out of the main list.

## Watchlist and near-misses

### Local Inverse Geometry Can Be Amortized

This is the strongest same-day adjacent hit. It learns a reusable reverse operator inside Deceptron and deploys it in D-IPG, an iterative inverse-problem solver. The loop is real: each solver step maps a residual-corrected measurement proposal back to latent space and updates the current estimate.

Why it was not added: the repeated computation is framed as an external solver trajectory for inverse problems, not as a learned layer/block/operator reused inside one model forward process. That makes it useful watchlist material, but not a conservative main-list addition under the repo's strict loop-model definition.

### KV-Fold: One-Step KV-Cache Recurrence for Long-Context Inference

KV-Fold treats the KV cache as an accumulator over sequence chunks. A frozen transformer processes each new chunk conditioned on the accumulated cache, appends new keys and values, and passes the enlarged cache forward. The reported behavior is stable across long chains and preserves needle-in-a-haystack retrieval up to 128K tokens.

Why it was not added: the recurrence is implemented as repeated chunk-level forward passes through a frozen model. It is relevant to cache recurrence and long-context systems, but the abstract does not make a same-forward reused internal learned block central.

### Inline Critic Steers Image Editing

Inline Critic adds a learnable token that critiques a frozen image-editing model at intermediate layers and steers subsequent hidden states during the ongoing forward pass. This is a useful example of within-forward refinement rather than post-hoc correction after full generation.

Why it was not added: the mechanism is an intervention token over ordinary model depth, not an explicit looped, weight-tied, recurrent-depth, or fixed-point operator.

### Elastic Attention Cores for Scalable Vision Transformers

VECA uses learned core tokens as a communication interface between patches, propagating those cores across layers and allowing compute/accuracy trade-offs along the core axis.

Why it was not added: the paper propagates learned state across layers, but the abstract does not claim reuse of the same learned layer, block, or operator. It is an adjacent efficient-attention design, not a strict loop model.

## Backlog note

**Visual Implicit Autoregressive Modeling** (`2605.01220`) looks much more clearly in scope because it embeds an implicit equilibrium layer and exposes a fixed-point iteration knob at inference. It was submitted on 2026-05-02, so it is kept out of this daily intake rather than mixed into the same-day watch. It is worth a separate backfill review if the maintainer wants to add older missed papers.
