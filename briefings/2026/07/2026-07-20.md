# Daily Loop-Model Briefing - 2026-07-20

## Outcome

Two recent papers were added today. Both sit squarely in the looped-Transformer thread, but they address different bottlenecks. **Loop the Loopies!** is a model-scaling paper: it presents Loopie MoE models and argues that a looped Transformer can outperform compute-matched vanilla Transformer baselines when the training recipe is adapted to repeated block reuse. **Looped Latent Attention** is a cache-compression paper: it targets the loop-indexed KV cache created by weight-tied recurrence and stores compact latents that reconstruct loop-specific K/V vectors when attention needs them.

## Added

### Loop the Loopies! (`2607.16051`)

Loopie is a looped Transformer series built with Mixture-of-Experts models. The paper frames the historical problem directly: spending extra compute on more parameters has usually beaten spending it on more loops. Loopie claims to narrow that gap by pairing looped MoE architecture choices with an adapted post-training pipeline.

Why it was added: the mechanism is direct reused-block recurrence. The paper studies looped Transformers, where learned Transformer blocks are applied repeatedly within one model forward process rather than through external prompting, agent calls, or repeated full-model invocations.

Reader takeaway: the abstract reports that Loopie beats vanilla Transformer baselines trained with the same compute budget, and that the post-trained model reaches strong reasoning performance on olympiad-style evaluations without tools.

### Looped Latent Attention: Cross-Loop KV Compression for Looped Transformers (`2607.15456`)

Looped Latent Attention focuses on a practical cost introduced by looped, weight-tied Transformers: decoding stores separate K/V cache entries for each recurrence step. The paper argues that, for a fixed token, layer, and head, K/V vectors across loops form a short low-rank trajectory. It then compresses this cross-loop structure and reconstructs loop-specific K/V vectors at attention time.

Why it was added: the paper is about the internal state of looped Transformers, not a generic cache trick. Its core object is the recurrence-indexed KV cache induced by reusing a learned block across loop steps.

Reader takeaway: the abstract reports better matched-budget performance than head-axis MLA, cross-layer sharing, KV quantization, and final-loop reuse, with transfer to Huginn-3.5B in decoder-independent evaluation.

## Checked but not added

- **Searching Videos as Trees** (`2607.16189`) trains an agent to navigate an adaptive temporal tree for video QA. The loop is agent search over video segments, not reused internal learned-module recurrence.
- **Recursive Harness Self-Improvement** (`2607.15524`) optimizes prompt/harness scaffolds over iterations. It is an external harness loop, not a looped model architecture.
- **An Exam for Active Observers** (`2607.16165`) is a benchmark for active visual observation in MLLMs. It tests repeated perception behavior but does not introduce a reused internal block or fixed-point model family.
- **Adaptive Multi-Step Lookahead Decoding for Diffusion Language Models** (`2607.15655`) improves diffusion-language-model decoding with lookahead. Diffusion/iterative decoding is outside the repo's strict loop-model boundary.
